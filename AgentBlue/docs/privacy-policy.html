개인정보 처리방침

최종 수정일: 2026년 2월 26일 · 시행일: 2026년 2월 26일

AgentDroid(이하 "본 앱")는 「개인정보 보호법」 제30조에 따라 이용자의 개인정보를 보호하고, 이와 관련한 고충을 신속하고 원활하게 처리할 수 있도록 다음과 같이 개인정보 처리방침을 수립·공개합니다.

제1조 (수집하는 개인정보 항목 및 수집 방법)

1. 본 앱은 서비스 제공을 위해 다음과 같은 정보를 수집합니다.

[자동 수집 항목]
  • 화면 UI 텍스트 정보 (접근성 서비스를 통해 수집되는 화면 요소의 텍스트, 힌트, 콘텐츠 설명)
  • 화면 UI 구조 정보 (뷰 ID, 패키지명, 클래스명, 요소 위치 및 상태)
[이용자 입력 항목]
  • AI API 키 (OpenAI, Google Gemini, Anthropic Claude, DeepSeek 등 사용자가 선택한 AI 서비스의 API 키)
  • 사용자 명령어 (에이전트에게 전달하는 자연어 명령)
  • 세션 코드 (Desktop 연동 시 입력하는 8자리 페어링 코드)
[서비스 운영 정보]
  • Firebase 익명 인증 UID
  • 실행 기록 — 기기 내 저장 (Room 데이터베이스):
    • 사용자 명령어, 실행 상태(완료/실패/취소), 결과 메시지
    • 전체 스텝 로그 (각 스텝의 액션 타입, 대상 텍스트, AI 추론 내용, 성공 여부)
    • 실행 시작 시간 및 종료 시간
  • 실시간 실행 상태 — Firebase Firestore 동기화 (Desktop 연동 시):
    • 현재 실행 중인 명령어, 진행 스텝, 최대 스텝 수
    • 각 스텝의 실시간 로그 (액션 타입, 대상 텍스트, AI 추론 내용, 성공 여부, 타임스탬프)
    • 에이전트 상태 (실행 중/완료/실패/취소/대기)

2. 수집 방법

  • Android 접근성(Accessibility) API를 통한 화면 정보 수집
  • 앱 내 사용자 직접 입력
  • Firebase 익명 인증 자동 생성
  • 에이전트 실행 중 자동 기록 (스텝 로그)

제2조 (개인정보의 수집 및 이용 목적)

본 앱은 수집한 정보를 다음의 목적을 위해 이용합니다.

  1. AI 기반 화면 분석 및 자동 조작: 접근성 서비스를 통해 수집한 화면 UI 정보를 AI 모델에 전송하여 사용자의 명령을 분석하고 적절한 동작(클릭, 입력, 스크롤 등)을 결정합니다.
  2. 명령 실행 및 결과 관리: 사용자가 입력한 명령의 실행 과정과 결과를 기기 내 데이터베이스(Room)에 기록하여 이전 실행 내역을 확인할 수 있도록 합니다. 저장 항목에는 명령어, 실행 상태, 각 스텝의 액션 로그(대상 텍스트, AI 추론 내용 포함), 시작/종료 시간이 포함됩니다.
  3. Desktop 연동 및 실시간 상태 공유: 세션 코드를 통해 데스크탑 앱과 연동하여 원격으로 명령을 전달받습니다. 세션이 연결된 상태에서 에이전트 실행 시, 현재 진행 상태와 각 스텝의 실시간 로그(액션 타입, 대상 텍스트, AI 추론 내용, 성공 여부)가 Firebase Firestore를 통해 데스크탑으로 동기화됩니다.

제3조 (접근성 API 사용에 관한 고지)

  1. 본 앱은 Android 접근성 서비스 API를 사용합니다. 이를 통해 현재 화면에 표시된 UI 요소(텍스트, 버튼, 입력 필드 등)의 정보를 읽고, 사용자를 대신하여 화면 조작(클릭, 텍스트 입력, 스크롤, 뒤로 가기 등)을 수행합니다.
  2. 접근성 API를 통해 수집되는 데이터:
    • 화면에 표시된 텍스트 및 힌트 텍스트
    • UI 요소의 콘텐츠 설명(contentDescription)
    • 뷰 리소스 ID 및 클래스명
    • 앱 패키지명
    • UI 요소의 위치(bounds) 및 상태(클릭 가능 여부, 편집 가능 여부)
  3. 접근성 API를 통해 수집된 데이터는 오직 AI 모델의 화면 분석 및 자동 조작 기능 수행을 위해서만 사용되며, 다른 목적으로 수집·저장·공유되지 않습니다.
  4. 접근성 API를 통해 수집된 화면 데이터는 기기에 영구 저장되지 않으며, AI 분석 요청 시 일시적으로 사용된 후 즉시 폐기됩니다.
  5. 본 앱은 접근성 API를 통해 수집한 데이터를 광고, 마케팅, 사용자 프로파일링, 또는 기타 접근성 기능과 무관한 목적으로 사용하지 않습니다.

제4조 (개인정보의 제3자 제공)

1. 본 앱은 다음과 같은 경우에 사용자의 정보를 외부 서비스로 전송합니다.

[AI API 서비스 제공자]
  • 제공받는 자: 사용자가 선택한 AI 서비스 제공자 (OpenAI, Google, Anthropic, DeepSeek 중 택 1)
  • 제공 항목: 현재 화면의 UI 트리 정보(텍스트, 구조), 사용자 명령어, 실행 기록 요약
  • 제공 목적: AI 기반 화면 분석 및 다음 동작 결정
  • 전송 방식: HTTPS 암호화 통신
  • 보유 기간: 각 AI 서비스 제공자의 API 데이터 처리 정책에 따름
[Firebase (Google)]
  • 제공받는 자: Google LLC (Firebase 서비스)
  • 제공 항목:
    • 익명 인증 정보(UID)
    • 세션 페어링 데이터 (세션 코드, 연결 상태)
    • 원격 명령어 및 실행 결과 (명령어, 처리 상태, 결과 메시지)
    • 에이전트 실시간 실행 상태 (현재 명령어, 진행 스텝, 각 스텝의 액션 타입·대상 텍스트·AI 추론 내용·성공 여부·타임스탬프)
  • 제공 목적: Desktop-Mobile 연동, 원격 명령 전달, 실행 상태 실시간 모니터링
  • 보유 기간: 세션 해제 시까지 (세션 해제 시 비활성화 처리)

2. 위 경우를 제외하고 이용자의 개인정보를 제3자에게 제공하지 않습니다.

제5조 (개인정보의 보유 및 이용 기간)

1. 기기 내 저장 데이터:

  • AI API 키: 사용자가 삭제하거나 앱을 제거할 때까지 (SharedPreferences)
  • 실행 기록: 사용자가 '기록 삭제' 기능으로 삭제하거나 앱을 제거할 때까지 (기기 내 Room 데이터베이스, 파일명: agentdroid.db). 저장 항목: 명령어, 실행 상태, 결과 메시지, 전체 스텝 로그(JSON), 시작/종료 시간
  • 에이전트 설정 및 동의 정보: 사용자가 변경하거나 앱을 제거할 때까지 (SharedPreferences)

2. 외부 전송 데이터:

  • AI API로 전송된 화면 데이터: 각 AI 서비스 제공자의 데이터 처리 정책에 따라 처리되며, 본 앱은 전송 후 해당 데이터를 보관하지 않습니다.
  • Firebase 실시간 실행 상태: 세션 연결 중 에이전트 실행 시 Firestore에 동기화되며, 세션 해제 시 비활성화 처리됩니다. 각 스텝의 대상 텍스트(targetText)에는 화면에서 조작한 UI 요소의 텍스트가 포함될 수 있습니다.
  • Firebase 세션 데이터: 세션 연결 해제 시 비활성화 처리

제6조 (개인정보의 파기 절차 및 방법)

  1. 파기 절차: 보유 기간이 경과하거나 처리 목적이 달성된 개인정보는 지체 없이 파기합니다.
  2. 파기 방법:
    • 기기 내 데이터: 앱 삭제(uninstall) 시 자동 삭제
    • 앱 내 '기록 삭제' 기능으로 실행 기록 삭제 가능
    • 세션 연결 해제 시 Firebase 세션 데이터 비활성화

제7조 (이용자의 권리·의무 및 행사 방법)

1. 이용자는 언제든지 다음의 권리를 행사할 수 있습니다.

  • 앱 설정에서 API 키를 삭제·변경할 수 있습니다.
  • 실행 기록을 삭제할 수 있습니다.
  • 세션 연결을 해제할 수 있습니다.
  • 접근성 서비스를 비활성화하여 화면 데이터 수집을 중단할 수 있습니다.
  • 앱을 삭제하여 모든 기기 내 데이터를 파기할 수 있습니다.

2. 동의 철회: 앱 내 설정에서 동의를 철회할 수 있으며, 철회 시 서비스 이용이 제한됩니다.

제8조 (개인정보의 안전성 확보 조치)

본 앱은 다음과 같은 안전성 확보 조치를 취하고 있습니다.

1. 기술적 조치:

  • 모든 외부 통신은 HTTPS(TLS) 암호화를 통해 이루어집니다.
  • API 키는 기기 내 SharedPreferences에 저장되며, 외부로 전송되지 않습니다. (AI API 인증에만 사용)
  • 화면 데이터는 AI 분석 요청 시 일시적으로만 사용되며 영구 저장되지 않습니다.
  • Firebase 익명 인증을 사용하여 최소한의 인증 정보만 관리합니다.

2. 관리적 조치:

  • 개인정보 처리에 관한 정책을 수립하고 시행합니다.
  • 접근성 API를 통해 수집된 데이터의 접근을 최소한으로 제한합니다.
